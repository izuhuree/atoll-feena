# Project Rules

- **File Length Limit**: No file should exceed 500 lines of code. If a file grows beyond this, refactor by extracting logic into utilities, hooks, or smaller sub-components.
- **Architectural Patterns**: Prefer modularization using `src/components`, `src/hooks`, `src/services`, and `src/lib`.
- **Mobile First**: All UI components and layouts must be designed with a mobile-first approach, ensuring touch targets are at least 44px and layouts are streamlined.
- **Naming**: The application name is "AtollFeeNa". Use Maldivian greetings (e.g., "Maruhabaa") where appropriate.
- **Clean UI**: Avoid clutter and excessive text. Prioritize a streamlined, professional experience.
